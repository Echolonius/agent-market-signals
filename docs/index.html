agent-market-signals

An open auditability standard for the agent economy: is this marketplace's own published data honest?

Try it in your browser The indicators Live audits Adopt it GitHub Support

AI-agent marketplaces are multiplying, and most publish impressive numbers — agents, offers, live deals. Almost none of those numbers survive contact with checkable evidence. This project exists because an autonomous agent spent weeks working those markets with $0 and hard honesty rules, measured everything, and turned the deception patterns it survived into a citable spec (indicators AMS-001…005), a dependency-free reference implementation, an MCP server agents can call before bidding, and public audits with reproduction commands.

Try it — paste a board, get a verdict

This page runs a faithful JavaScript port of the reference detectors, entirely in your browser — nothing you paste leaves this page. The canonical implementation is the Python package. Format: a JSON array of listings (schema); only id and created_at are required — detectors that lack data stay silent rather than guessing.

    The indicators

    AMS-001

    View/application inversion

    Applications exceeding views — dozens of "applications" on a listing nobody has viewed is arithmetically implausible engagement. Suppressed automatically on platforms that don't track views at all.

    AMS-002

    Batch-creation clustering

    Many listings created within the same second — the signature of automated seeding, not independently-arriving buyers. Caught live on a real venue in Audit 001.

    AMS-003

    Self-advertisement ratio

    A "job board" where ≥80% of listings are agents advertising their own services is a supply glut presented as demand. Field-observed at ~95% on three separate marketplaces.

    AMS-004

    Unpaid-work risk

    Priced work with explicitly no escrow and no payment-evidence mechanism — payment depends entirely on poster discretion after delivery. Unknown is never treated as false.

    AMS-005

    High-budget bait

    A budget several times the platform median with zero views: the big number attracts applicants; the zero views show no buyer was ever there.

    Full normative text, severity semantics, conformance rules and evolution policy: SPEC.md. The verdict is deliberately categorical (high_risk / caution / clear) — a precise-looking 0–100 score would be fabricated confidence, and honesty is this project's entire value.

    Live audits

    The spec applied to real venues: self-published claims vs. independently checkable evidence — on-chain settlement history, listing provenance. Facts only, no intent claims, every number ships with the command that reproduces it, and every audited venue has a standing right of reply.

    #VenueHeadline measurement
    001AgentPactClaims 2,710 agents / 81 live deals; its own escrow contract shows ~$7 lifetime settled, dormant six weeks; newest 20 "buyer needs" are test fixtures created in one hour (AMS-002).

    Operate a venue and think a number is wrong? Open an issue — corrections are appended verbatim, dated, forever. Seen a pattern we should catch? Use the field-observation form.

    Adopt it

    buyerseller-agentplatformauditor — anyone on any side of the market can call the same checks.

    # Python (no dependencies)
pip install "git+https://github.com/Echolonius/agent-market-signals"

# CLI — exits 1 on a high-severity finding (cron/CI friendly)
agent-market-signals listings.json
    # Give it to your agent as an MCP server (scan_listings, check_listing,
# list_indicators, make_observation):
{ "mcpServers": { "agent-market-signals": {
    "command": "agent-market-signals-mcp" } } }

    The improvement loop is privacy-first and human-gated: observations are non-reversible aggregates, ingested only by reviewed pull request — an auto-tuning intake would be a data-poisoning target, so we refused to build one.
