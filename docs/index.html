Is that marketplace
lying to you?

Agent-economy job boards publish big numbers — agents, offers, “live deals.” Most don’t survive contact with checkable evidence. This is an open standard, born from weeks of field work, for auditing a marketplace’s own published signals — before you spend labor there.

Everything below runs in your browser. What you paste is parsed as data (JSON.parse), never executed, never uploaded.

Machine-readable index: llms.txt. Everything on this page is callable as a library, CLI, or MCP server.

Check a board in 30 seconds

Paste a JSON array of listings (schema — only id and created_at required; checks that lack data stay silent rather than guessing).

    What it looks for

    AMS-001

    Applications without views

    Dozens of “applications” on a listing nobody has viewed is arithmetically implausible engagement. (Auto-suppressed on platforms that don’t track views.)

    AMS-002

    Batch-created listings

    Many listings created in the same second is automated seeding, not independently-arriving buyers. Caught live in Audit 001.

    AMS-003

    A board that’s all sellers

    When ≥80% of “jobs” are agents advertising their own services, supply glut is being presented as demand. Field-observed at ~95% on three marketplaces.

    AMS-004

    Priced work, no way to get paid

    A budget with explicitly no escrow and no payment-evidence mechanism means payment depends entirely on the poster’s goodwill after delivery.

    AMS-005

    High-budget bait

    A budget several times the platform median that nobody has viewed: the big number attracts applicants; the zero views show no buyer was ever there.

    Verdicts are deliberately categorical — high_risk / caution / clear. A precise-looking 0–100 score would be fabricated confidence; honesty is the product. Full normative text: SPEC.md.

    Public audits

    Wire it into your agent

    Three surfaces, one core. Dependency-free Python 3.9+. Unknown fields are never treated as false; detectors that lack data stay silent.

    1 · MCP server (recommended — call it before you bid)

    pip install "agent-market-signals[mcp] @ git+https://github.com/Echolonius/agent-market-signals"

# claude_desktop_config.json / .mcp.json
{ "mcpServers": { "agent-market-signals": { "command": "agent-market-signals-mcp" } } }

    Tools exposed: scan_listings (vet a whole board) · check_listing (vet one listing before bidding) · list_indicators (AMS catalog) · make_observation (opt-in, privacy-preserving aggregate — see FLYWHEEL.md).

    2 · Library

    pip install "git+https://github.com/Echolonius/agent-market-signals"

from agent_market_signals import Listing
from agent_market_signals.detectors import scan

result = scan([Listing.from_dict(item) for item in board_json])
result["verdict"]     # "high_risk" | "caution" | "clear"
result["findings"]    # list of {indicator, severity, listing_id, detail}
result["coverage"]    # how much data was actually assessable

    3 · CLI (cron/CI — exits 1 on a high finding)

    agent-market-signals listings.json

    Decision policy suggestion

    # before investing labor in any marketplace listing:
verdict == "high_risk"  -> do not bid; log the finding
verdict == "caution"    -> require payment evidence before work
verdict == "clear"      -> proceed, but check coverage — thin data flags little
